Sprint 02·The Loop
A Field Guide to Generative AI
Six stages · continuous loop

Perceive → Reason → Plan → Act → Observe → Reflect.

01 · Perceive
Receive input
user message, API trigger, scheduled event, sensor data
02 · Reason
Interpret
LLM reads input in context of memory, tools, and goal
03 · Plan
Decide sequence
what to do first, second, and in what order
04 · Act
Execute
calls a tool, writes to a DB, sends a message
05 · Observe
Receive result
success, failure, or partial data returns to LLM
06 · Reflect
Evaluate
goal met? if not — loop back to Reason with new context
A great consultant doesn't give up when their first call fails. They try the next contact. The loop is that instinct — built in.

Sprint 05·Planning
A Field Guide to Generative AI
Three strategies · same task · different paths

ReAct · Chain-of-Thought · Self-Reflection.

ReAct
Think → Act → Observe → Think → Act…
Interleaves reasoning with real tool calls. Adapts at each step based on what the tools return.
Best for: tool-heavy tasks, live data needed. Example: research a topic with iterative web search.
Chain-of-Thought
Step 1 → Step 2 → Step 3 → Answer
Pure reasoning chain — no tools called. Works through complexity using the LLM's own knowledge.
Best for: complex analysis, multi-variable decisions. Example: evaluate 3 vendor proposals against 6 criteria.
Self-Reflection
Draft → Critique → Redraft → Finalize
The agent critiques its own output and iterates until quality threshold is met.
Best for: high-stakes outputs — reports, proposals, legal. Example: draft → critique → redraft a contract clause.
3-question selector: Task complexity (simple → CoT · live data → ReAct · open-ended high-stakes → Reflection) · Tool dependency (needs tools → ReAct · internal only → CoT/Reflection) · Output quality (high-stakes → Reflection · speed → ReAct)

Design the governance layer for your agent.

Think of a real agent you'd deploy at work. Use the Claude prompt in your workbook to generate a first draft manifest. Narrow every scope. Find the nervous tool.

The first draft is always too broad on scope and too permissive on permission. Narrow both. Then circle the tool that makes you nervous.

Sprint 06·Multi-Agent
A Field Guide to Generative AI
Three patterns · when one agent isn't enough

Orchestrate · Parallelize · Hand off.

Orchestrator-Worker
Project manager + specialists
Orchestrator→ Worker A· Worker B· Worker C→ Synthesis
Best for: complex tasks with sequential dependencies. Example: market research → web + finance + competitor → unified brief.
Parallel Agents
Multiple analysts, shared queue
500 items→ Agent 1· Agent 2· Agent 3→ Merge
Best for: independent sub-tasks, speed matters. Example: process 500 resumes in parallel → merged ranking.
Specialist Handoffs
Relay race — each runner specialized
Ticket→ Classify→ Resolve→ Escalate
Best for: distinct sequential phases needing different expertise. Example: your S2 HR Screener — search → extract → score.
