Concept 2·The Loop Recovers
A Field Guide to Generative AI
Overdue-invoice agent · one email bounces

When a step fails, the loop doesn't quit — it tries another way.

A chatbot would send the batch, hit the bounce, and stop with "1 failed." The agent treats the bounce as new information, loops back, and finds another route. Follow the dots:

Perceive
8 a.m. trigger fires: "run the overdue check"
Plan
Pull the list, draft a reminder, send each — 14 invoices
Act
Sends all 14 reminder emails
Observe
13 delivered, 1 bounced — that address is dead
Reflect
"Goal not met for one. The email on file is stale."
Done
14 / 14 delivered
↺ loop backInstead of stopping, it re-plans: look up a current contact in the CRM → re-send to the new address → check again. The bounce became the trigger for a better plan, not a dead end.
But a loop needs a stop sign. Three exit conditions, all required: goal achieved · max tries reached · human escalation. Without them, an agent that can't recover just loops forever and burns money.

Concept 2·The Gap, Drawn
A Field Guide to Generative AI
Your HR Screener · the same loop, no stop sign at Reflect

The tool came up empty. With no guardrail, the loop bluffed.

Same six stages as the invoice agent — but here the tool can't succeed (no way to reach real LinkedIn data), and there's no refusal rule at Reflect. Watch where the two agents diverge at the final stage:

Perceive
Job params: title, location, years
Plan
"Get 5 matching profiles via search"
Act
Web search runs — but can't reach LinkedIn
Observe
Nothing valid comes back
Reflect?
no rule here to decide what to do with an empty result
✕ What the screener did (no guardrail)
Reached the gap, found nothing, and filled it with plausible fake LinkedIn URLs. The loop ran to the end — it just had nothing true to end with.
✓ What a guardrail would have done
A refusal rule at Reflect: "if no verified profiles, say so and escalate to a recruiter — never invent." Same loop, honest ending.
Root cause: the tool (Gap #3) — there's no legal way to fetch real LinkedIn profiles. The missing safeguard: a refusal guardrail — so the gap ends in "I can't verify these," not invention. Tool failed; guardrail was absent; hallucination was the result.

Concept 5·Planning
A Field Guide to Generative AI
3 strategies · same task, different shapes

3 ways an agent can think.

ReAct
Best for: tool-heavy, live-data tasks
Think, do, see, think again — improvises with live data as each result lands.
Lumière"Should we run a delivery promo this weekend?" → checks the weather tool, then the order backlog, then decides.
Chain-of-Thought
Best for: analysis from known facts
A pure reasoning chain — no tools, just careful step-by-step thinking.
Lumière"Which 3 cakes should we feature this month?" → weighs margin, prep time, and season, then recommends.
Self-Reflection
Best for: high-stakes writing
Draft, critique its own draft, redraft — until it's good enough to send.
LumièreAn apology message to a customer whose cake was late → drafts, critiques the tone, rewrites before sending.
Pick by 3 questions: how complex is the task · does it need live tools · how high are the stakes. Answer those and the right strategy picks itself.
