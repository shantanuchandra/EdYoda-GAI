Order Assistant · tool manifest

Five tools. Each closes Gap #3 (Tools) or #4 (Autonomy).

01orderRulesReaderREADReads Lumière's KB — lead times, sizes, eggless, allergens.
02slotBookerREAD+WRITEProduction calendar only. Returns booking ID.
03confirmationSenderSENDTemplated WhatsApp message. Never freeform.

Regulation Monitor · tool manifest

Five tools. Closes Gap #1 (Memory — Seen-Rules Log) + Gap #5 (Planning).

01fetchRulesREADPolls the FSSAI feed (simulated here).
02classifyRiskREADSorts into buckets the human pre-defined. Never invents a bucket.
03autoApplyWRITECosmetic rules only. Logs the change.
04escalateToHumanSENDAllergen / ingredient / safety → human gate.
05seenRulesLogREAD+WRITEEpisodic memory. localStorage. Never re-alerts.

How this maps to the workbook

Every concept from S5 is live here:

  • The 6-stage loop — visible in every Agent steps trace.
  • Four tool families — readers, writers, senders, computers — all five Order Assistant tools mapped.
  • Episodic memory — the Seen-Rules Log. Refresh the page; it remembers.
  • Risk bucketing — the human pre-defines buckets, the agent sorts. Allergen / safety always escalates.
  • S2 hallucination cure — every tool has a narrow scope; the bot refuses cleanly when there's no tool for the job.
