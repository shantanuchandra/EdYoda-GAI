A Field Guide to Generative AI
Concept sprint 3 of 4

The IF node: where the agent's answer becomes a decision.

One condition. Two paths.

The agent returns human_gate: true or false. The IF node reads it and routes the workflow — machine-speed left, human review right.

You've been doing this your whole career. "If the deal is above ₹10L, escalate to the VP." That's an IF node. Today n8n runs it for you.

S6·Sprint 4 · Two Ears
A Field Guide to Generative AI
Concept sprint 4 of 4 — the hardest one

Two Ears: the Human-in-the-Loop pattern.

Ear 1 — Gmail Trigger
Order email arrives
Workflow wakes up and starts running
↓
Wait node
Workflow sleeps
Not cancelled. Not deleted. Holding — until the second ear hears something.
↓
Ear 2 — Telegram Trigger
Manager taps ✅ or ❌
Workflow wakes up again — only on the right branch

The human doesn't need to log into n8n.

They get a Telegram message with two buttons. That's the entire interface. The machine does the work. The human makes the call. The machine finishes the job.

This pattern appears in every serious automation: invoice approvals, content sign-offs, medical referrals, legal reviews.

Build Step B · Add the Human Gate

Step 2 — Wait node (the workflow goes to sleep)

Add a Wait node after the Telegram Send. Mode: On webhook call.

n8n generates a Resume URL automatically. The workflow stops here — holding until the Telegram Trigger calls that URL.

The film metaphor
Pause, not stop.

The frames are still in memory. When the Telegram callback fires, it presses play from exactly where it stopped.

✦ n8n AI Prompt (S3 RCTFC)
ROLE: n8n builder, continuing Lumière automation. CONTEXT: After Telegram send, workflow must pause until manager taps a button. TASK: Add a Wait node after Telegram. Set to resume on webhook — waits indefinitely. FORMAT: Name "Wait for Manager Decision". This is the bridge between the Two Ears. CONSTRAINTS: Resume mode = webhook · no timeout · do not set a time delay.

S6·Build C · Pin Data
A Field Guide to Generative AI
Technique

Test without sending real emails.

Pin data from a previous run. The Trigger replays the pinned email instead of polling Gmail.

1
Open Gmail Trigger output from any successful run
2
Click Pin data
3
Run workflow — Trigger uses pinned email. No Gmail polling, no inbox clutter.

Use this for every test after your first one. Faster and cleaner.
