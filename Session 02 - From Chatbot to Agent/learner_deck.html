Block 1 · Concept Sprint·Sprint 01

Sprint 01 · Core concept

The five gaps agents close.

What chatbots cannot do — and how autonomous AI agents fix each limitation. Ordered by severity. Numbered in the order we'll close them as a class today.

#1Memory gap Chatbot forgets everything after the session ends. Agent persists context across runs, hours, days. Critical
#2Multi-step reasoning Chatbot handles one instruction at a time. Agent manages chains of dependent decisions. Critical
#3Tools gap Chatbot only produces text. Agent calls APIs, reads files, writes to real systems. High
#4Autonomy gap Chatbot waits for input on every step. Agent continues until the goal is complete. High
#5Planning gap Chatbot answers one question. Agent decomposes goals into ordered steps with a time horizon. Medium

Each gap is a class of work your chatbot silently refuses to do. Naming them is half the fix — closing them is the rest of today. Watch for Gap #3 in Step A, Gap #4 in Step C, Gap #1 in Step D, and Gaps #2 + #5 in Step E.

Sprint 04 · The discipline

The four-question funnel — chatbot, RAG, agent, multi-agent.

— Ask in order. One survives. —
1
Does the answer exist in a fixed document or database?
Yes → RAG No → continue
2
Is one response — with no follow-up action — enough?
Yes → Chatbot No → continue
3
Does it require multiple steps, memory, or tool use?
Yes → Agent No → continue
4
Does it need multiple specialised capabilities in parallel?
Yes → Multi-Agent
— The four shapes —
RAG Document retrieval. Grounded answers from a fixed corpus.
Chatbot Single response. One question, one reply, no follow-through.
Agent Multi-step tasks. Plans, uses tools, persists memory.
Multi-Agent Parallel execution. Specialists orchestrated toward one goal.

Three questions eliminate. Only one path survives. Most teams skip this discipline and overbuild — 95% pattern-matching success when you don't.
