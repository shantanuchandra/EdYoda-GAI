# Session 4 — RAG: Giving Agents a Brain of Your Own Data — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce the four Session 4 learning artifacts (facilitator script, learner workbook, learner deck, presenter deck) for the "GenAI & AI Agents for Non-Coders" course, grounded in the approved design spec.

**Architecture:** Content artifacts, not software. Four files built in README order (script → workbook → learner deck → presenter deck) from the `templates/` scaffolds. The script is the source of truth; the workbook mirrors its step labels; the two decks are a *symmetric pair* (identical slide count, 1:1 index alignment) wired through `localStorage`. Carousel is deferred (built post-delivery as a recap). This plan specifies **what content goes in each slot and how to verify it** — it does not contain literal HTML/JS/Markdown code; the builder writes that from the templates following the specs below.

**Tech Stack:** Markdown (script + workbook), self-contained HTML/CSS/JS (decks, fonts via Google CDN, no build step). Verification via `grep`, `python3 -m http.server` + browser, and `pdftoppm` for visual render checks.

**Source documents (read before starting):**
- Spec: `docs/superpowers/specs/2026-05-29-session-04-rag-design.md` — the authority for all content decisions.
- Templates: `templates/README.md` + the five `*.template.*` files — the authority for all design/structure decisions.
- Reference deck (mine for content, do NOT copy framing): `Session 04 - RAG Giving Agents a Brain of Your Own Data/Deck-RAG.pdf`.
- Prior session for tone/structure: `Session 03 - Prompt Engineering Context Engineering for Agents/01_Facilitator_Script.md`.
- Memory canon: all files referenced in `MEMORY.md`, especially `edyoda_session_boundaries`, `edyoda_presenter_vs_learner`, `edyoda_facilitation_patterns`, `feedback_ground_before_claiming`, `edyoda_author_colophon`.

**Cross-cutting invariants (apply to EVERY task — re-check before each commit):**
- **Two modes never mixed:** dark/teal = presenter; paper/terra = learner deck. Copy `:root` verbatim from templates.
- **Fonts fixed:** Inter + JetBrains Mono (dark); Fraunces + Geist + Geist Mono (paper).
- **Hallucination is the *premise*, not a teaching unit** (per spec §1, §5, §10): one motivating line, one ≤60-sec live proof, one honesty note on fine-tuning. Never dwell.
- **Types of RAG ("the dial") is first-class** (spec §6): Level 0 No-retrieval → 1 CAG (=Build A) → 2 Vanilla (=Builds B–D) → 3 Hybrid (=Build E) → 4 Agentic (=S5 bridge). One worked example each, on the running doc.
- **`say:` fields = spoken English.** No timestamps/stage directions on learner-facing artifacts. Bridge to S5 only on the close slide.
- **Vector DBs (Pinecone/Weaviate) = concept layer only**, framed "to brief your eng team," never wired today.
- **Author colophon** (Shantanu Chandra + linkedin.com/in/chandrashantanu) closes every artifact.
- **Not a git repo.** "Commit" steps are written as `git` commands per template convention, but if `git status` errors, SKIP the commit step and note "no-VCS: file on disk is the record" — do not `git init`.
